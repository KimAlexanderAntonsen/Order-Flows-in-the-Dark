# Urgency Variable Construction

The two construction blocks build the firm-specific earnings urgency panel and
the market-wide macro urgency panel that later enter the VARX in `04_VARX`
as predetermined exogenous regressors.

## 1. Earnings urgency (Menkveld-style)

Source script: `04_construct_earnings_urgency.py`
Output: `data_clean/earnings_urgency_sparse_panel.csv`

The earnings urgency variables follow the construction in Menkveld, Yueshen,
and Zhu (2017). The output is a stock-minute exogenous panel for every firm
in the corrected 2019 S&P 500 universe over the thesis sample window.

### Construction

The earnings surprise is the absolute difference between announced EPS and
expected EPS, scaled by the closing price on the business day immediately
before the announcement:

$$
EA\_Surprise_a = \frac{\left| EPS^{reported}_a - EPS^{expected}_a \right|}{P^{close}_{a,pre}}
$$

Earnings announcements that fall outside trading hours are mapped onto the
**next trading day** through 13 intraday variables, one for each regular-hours
30-minute block:

- `post_ea_1` for 09:30-10:00
- `post_ea_2` for 10:00-10:30
- ...
- `post_ea_13` for 15:30-16:00

Each earnings event is expanded into minute-level variables that hold the
event's earnings surprise inside the relevant 30-minute block on the effect
trading day and zero elsewhere.

### Raw earnings event input

The raw earnings event file is generated by `03_fetch_nasdaq_earnings.py`,
which pulls the historical Nasdaq earnings calendar. The source provides
announcement date, actual EPS, and consensus EPS forecast. When the
historical announcement time is reported as `time-not-supplied`, the
construction treats the event as outside regular hours and assigns the
earnings surprise to the next trading day's 13 regular-hours blocks.

The construction script reads `data_raw/earnings_events.csv` and requires at
minimum:

- `ticker`
- `announce_datetime`
- `reported_eps`
- `expected_eps`

Additional source columns are tolerated. The fetcher writes this file.

## 2. Macro urgency

Source script: `06_construct_macro_news_urgency.py`
Helper module: `05_macro_urgency_helpers.py`
Output: `data_clean/macro_news_minute_panel.csv`

The macro urgency variable covers two narrow U.S. event groups:

- U.S. inflation releases: `CPI` and `PPI`
- U.S. interest-rate news: scheduled `FOMC` statement / rate-decision releases

Following Menkveld, the construction uses minute-level announcement dummies
rather than aggregating heterogeneous macro surprises into a single scalar
shock. The script writes:

- combined dummies: `pre_news_1min`, `post_news_0min`, ..., `post_news_4min`
- inflation-specific dummies: `pre_inflation_1min`, `post_inflation_0min`, ..., `post_inflation_4min`
- rate-specific dummies: `pre_rate_1min`, `post_rate_0min`, ..., `post_rate_4min`

The 24 (two inside exclusion window) release timestamps inside the sample window are held inline in
`05_macro_urgency_helpers.py` (`CPI_DATES`, `PPI_DATES`, `FOMC_DATES`).
`build_macro_news_events()` materialises them as a DataFrame.

### Event-timestamp convention

The announcement time is stored in New York local time. Minute-bar rows are
interpreted as end-of-minute observations, so the event anchor used for the
macro dummies is the first minute **after** the scheduled release time:

- an `08:30` inflation release maps to `08:31` as `post_news_0min`
- a `14:00` FOMC statement maps to `14:01` as `post_news_0min`

The clean output stores two timestamp variants:

- `anchor_timestamp`: New York clock time with timezone removed
- `anchor_timestamp_legacy_varx`: shifted timestamp aligned with the VARX
  panel's `timestamp_legacy_varx` column

### Saved outputs

The macro construction writes two files:

- `data_clean/macro_news_events_clean.csv` (timestamped events used by the panel)
- `data_clean/macro_news_minute_panel.csv` (minute-level dummy panel)

The minute panel is common across stocks and is merged into the stock-minute
VARX panel as a market-wide exogenous regressor set.

## Run order

A full rebuild of the upstream urgency inputs runs the three scripts below in
order. This is not part of the normal rerun path described in `00_README.md`;
it applies only when the upstream raw data needs to be regenerated.

1. `03_fetch_nasdaq_earnings.py` -> `data_raw/earnings_events.csv` and `data_clean/earnings_events_coverage.json`
2. `04_construct_earnings_urgency.py` -> `data_clean/earnings_urgency_sparse_panel.csv`
3. `06_construct_macro_news_urgency.py` -> `data_clean/macro_news_events_clean.csv` and `data_clean/macro_news_minute_panel.csv`
